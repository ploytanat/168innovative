# SEO Keyword Map

This map is based on the current route structure in the repo and should be used to separate search intent across category, product, and article pages.

## Rules

- Category pages own broad commercial keywords.
- Product pages own model-specific and size-specific keywords.
- Article pages own informational keywords and must link into category and product pages.
- Do not make product pages compete for the same broad keyword as their parent category page.

## Internal Linking Model

- Article -> Category:
  - informational query should pass relevance to the main commercial hub
- Category -> Product:
  - broad intent page should link to the best exact-match SKUs and models
- Product -> Category:
  - every model page should point back to the parent category
- Product -> Article:
  - use 1 relevant educational article to support topical depth

## Content Priorities

### Category pages

- Expand intro copy to 250-500 words.
- Mention target keyword in the first paragraph.
- Add use cases, industries, OEM angle, and selection guidance.
- Link to 3-6 priority product pages.

### Product pages

- Lead with model + size + use case.
- Add 150-300 words of unique content beyond specs.
- Mention where the model fits compared with nearby sizes.
- Keep parent category and related model links visible.

### Article pages

- Write articles around comparison, sizing, and selection intent.
- Include strong CTAs back to category and product pages.
